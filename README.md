# 麻将强化学习策略研究

用强化学习（PPO + 自我博弈）训练麻将 AI，寻找自定义规则下的最优打法策略。

这是一个玩具项目，起源于对家里「村规麻将」的研究兴趣。项目的主要目的有两个：一是探索 Claude Code 的辅助编程能力，二是借此实践机器学习的相关知识。规则本身并非标准麻将，而是融合了家庭自定义玩法，不具有普适性。

---

## 游戏规则

**基本设置**
- 3人游戏，136张牌（万/筒/条各9×4，字牌7×4），无花牌
- 仅支持自摸胡牌，不支持点炮
- 支持碰、杠（暗杠/明杠/加杠），不支持吃

**万能牌（癞子）**
- 每局开始时掷骰子随机确定，全场4张同种牌均为万能牌
- 万能牌可代替任意牌（十三幺除外）
- 碰万能牌后只能胡「集齐4张万能牌」；杠万能牌直接胡牌

**胡牌门槛**
- 底分 ≥ 10分方可胡牌
- 若本局曾持有2张及以上万能牌，门槛提高至 ≥ 20分

**计分**

| 牌型 | 底分 |
|------|------|
| 清一色 | 10分 |
| 一条龙（1-9同花色）| 10分 |
| 七对 | 10分 |
| 三调一（七对中某牌凑满4张）| 20分 |
| 字一色 | 20分 |
| 十三幺 | 100分 |
| 集齐4张万能牌 | 40分 |
| 第1/2/3个杠 | +10/15/20分（累计）|
| 杠上开花 | 所有分 ×2 |
| 庄家 | 输赢均翻倍 |

## 项目结构

```
env/
  game.py            # 游戏核心逻辑（发牌、动作、结算）
  majiang_env.py     # Gymnasium 环境接口、观测空间、动作掩码
  single_agent_env.py# 单智能体包装、奖励塑形
  hand.py            # 胡牌检测（标准型/七对/清一色/一条龙/十三幺）
  shanten.py         # 向听数计算
  scorer.py          # 计分与结算
  stats_callback.py  # 训练统计回调
agents/
  random_agent.py    # 随机基准智能体
train.py             # 单次训练脚本
train_until.py       # 自我博弈循环，训练至目标分
inspect_strategy.py  # 策略分析工具
```

## 训练方法

**算法**：MaskablePPO（带动作掩码的近端策略优化）

**自我博弈**：维护历史 checkpoint 池（最近5个），每轮从池中随机采样对手，避免策略单一和震荡。

**奖励塑形**：
- 向听数改善奖励（鼓励向听牌方向发展）
- 听牌奖励（向听数=0时额外加分）
- 杠牌即时奖励 + 听牌状态下加杠额外奖励（激励杠上开花）
- 流局额外惩罚（在结算分基础上扣分，抑制消极流局）

**启动训练**
```bash
pip install -r requirements.txt
nohup python3 train_until.py --target 8 --n_envs 8 >> logs/train_until.log 2>&1 &
```

## 关于本项目

本项目由项目作者主导，[Claude](https://claude.ai)（Anthropic）辅助完成。

**项目作者的贡献**

这个项目的核心价值来自于作者对麻将领域的深入理解。作者设计了完整的自定义规则体系——包括万能牌机制、胡牌门槛、杠上开花结算等细节，这些规则的合理性和完整性是项目能够成立的前提。

在整个研究过程中，作者持续主导方向：发现训练震荡并判断根本原因、识别智能体策略的合理性问题（如「能胡不胡」、开会占比过高、碰牌率异常）、在 AI 给出错误建议时及时纠正（如关于万能牌应握在手中的策略判断）。关键的超参数选择、奖励设计方向、训练策略的取舍，均由作者决策。

**Claude 的贡献**

Claude 负责了代码实现的主体工作：游戏环境、向听数计算、胡牌检测、奖励塑形、自我博弈训练循环、统计回调等模块均由 Claude 编写。训练过程中的若干 bug（评估分被中间奖励污染、暗杠信息泄露给对手观测、统计写入频率等）也由 Claude 定位并修复。

值得客观指出的是：Claude 在部分策略判断上存在偏差，例如错误建议惩罚持有万能牌的行为，被作者纠正。Claude 的角色更接近一个执行力强但需要方向指引的工程师，而非研究的主导者。
